Insufficient Logging and Monitoring

The importance of securing a website cannot be overstated enough. That being said, it is also not realistic to aim for 100% security as there are always bound to be some vulnerabilities and unsecure end points in oneâ€™s web applications. The best possible course of action that can be adopted by any organisation is to keep monitoring the application and to take prompt action in case of any security breach or attack. Insufficient logging and monitoring can be dangerous as attackers rely on the lack of monitoring cycles and delayed responses to achieve their goals. Thus keeping audit logs are important for any web application. Audit logs are documents that record events that take place in a web application. These logs allow to look for and spot anomalies and hence take prompt actions to prevent large scale harmful attacks on the system.

To prevent security breaches owing to the lack of logging and monitoring, the following suggestions should be adopted: All login and data manipulation events and attempts should be logged into the systems central audit along with the associated user account details. These details should also be held in the memory of the system for sufficient amount of time so as to perform forensic operations in case of attacks. Generated logs should be present in an easily consumable format so that they can suitably be manipulated by the centralized log management solutions. High value transactions should have an audit trail with integrity controls to prevent illegal operations. Effective monitoring and alerting systems should be established so that suspicious activities are detected and dealt with timely. A suitable and robust response and recovery plan should be established well in advance.
